# Тестове середовище

Ця директорія містить всі автоматизовані тести проекту.

Документація, що описує стратегію, архітектуру та стандарти написання тестів, знаходиться в основній документації проекту.

➡️ **[Читати повний посібник з тестування](../../docs/18-testing/README.md)**

---

## Швидкий старт: Запуск Тестів

Нижче наведено основні команди для роботи з тестами.

- **Запустити всі тести (unit + integration):**
  ```bash
  npm test
  ```

- **Тільки Unit-тести (швидко, без БД):**
  ```bash
  npm run test:unit
  ```

- **Тільки Integration-тести (з підключенням до тестової БД):**
  ```bash
  npm run test:integration
  ```

- **Режим спостереження (автоматичний перезапуск при змінах):**
  ```bash
  npm run test:watch
  ```

- **Звіт про покриття коду:**
  ```bash
  npm run test:coverage
  ```

- **Запуск у візуальному UI режимі:**
  ```bash
  npm run test:ui
  ```

## Структура директорії

- `/unit`: Швидкі юніт-тести для ізольованої логіки.
- `/integration`: Інтеграційні тести, що взаємодіють з тестовою БД.
- `/lib`: Допоміжні утиліти, конфігурації та фікстури для тестів.
